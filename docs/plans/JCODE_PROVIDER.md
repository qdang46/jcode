# JCODE_PROVIDER — Master Plan

> Rebuild jcode provider system to opencode's Catalog → Integration → Credential architecture
> Generated by ultracode across 3 reference repos (opencode, oh-my-pi, CCB) + jcode current state deep-dive

## 2. Target Architecture

```
                    ┌─────────────┐
                    │   Config    │ user.toml + project.toml
                    └──────┬──────┘
                           │ --provider, --model, provider_profile
                           ▼
  ┌──────────────────────────────────────────────────────┐
  │                    CATALOG                            │
  │  providers: Map<ProviderID, ProviderRecord>           │
  │  models:    Map<ProviderID, Map<ModelID, ModelInfo>>  │
  │  default:   Option<(ProviderID, ModelID)>             │
  │                                                       │
  │  .available() → providers with credentials            │
  │  .default()  → best available model                   │
  │  .small()    → cheapest available model               │
  └──────┬───────────────────────────────────────────────┘
         │ lookup
         ▼
  ┌──────────────────────────────────────────────────────┐
  │                  INTEGRATION                          │
  │  providers: Map<ProviderID, LoginProvider>            │
  │  connections: Map<ProviderID, ConnectionInfo>         │
  │                                                       │
  │  .oauth(provider, method) → Attempt (10-min TTL)     │
  │  .key(provider, api_key) → save to credential store   │
  │  .connection.for(provider) → OAuth | Key | Env | None │
  └──────┬───────────────────────────────────────────────┘
         │ persist
         ▼
  ┌──────────────────────────────────────────────────────┐
  │               CREDENTIAL STORE                        │
  │  SQLite-backed (via jcode-keyring-store)              │
  │  CredentialTable: id, integration_id, label, value    │
  │                                                       │
  │  OAuth { access_token, refresh_token, expires_at }    │
  │  ApiKey { key }                                       │
  │                                                       │
  │  .create() → deletes old, inserts new (tx)            │
  │  .list(provider_id) → all credentials                 │
  │  .remove(id) → revoke                                 │
  └──────────────────────────────────────────────────────┘

  At runtime: Route Resolution
  ┌──────────────────────────────────────────────────────┐
  │  1. Agent::new()                                     │
  │  2. resolve_model() → Catalog.default() | config     │
  │  3. check_credentials() → Integration.connection()    │
  │  4. build 4-axis Route                                │
  │     Route { Protocol, Endpoint, Auth, Framing }      │
  │  5. send via Transport (HTTP/WS)                      │
  └──────────────────────────────────────────────────────┘
```

### Phase 1: Credential Store — SQLite-backed (2-3 days)

**Goal:** Replace in-memory CredentialStore stub with real SQLite persistence.

**SQLite schema:**
```sql
CREATE TABLE provider_credentials (
    id TEXT PRIMARY KEY,
    provider_id TEXT NOT NULL,
    label TEXT NOT NULL DEFAULT 'default',
    value TEXT NOT NULL,  -- JSON { type: "oauth", ... } or { type: "key", ... }
    time_created INTEGER NOT NULL,
    UNIQUE(provider_id, label)
);
```

**Behaviors (from opencode `credential.ts`):**
- `create()`: wrap in transaction — DELETE old for same provider_id, INSERT new
- `list(provider_id)`: return all, newest first
- `get(id)`: single credential by ID
- `update(id, updates)`: partial update (label/value)
- `remove(id)`: revoke

**Encryption:**
- Optional: encrypt `value` at rest via `jcode-keyring-store` (macOS Keychain / Linux Secret Service / Windows Credential Manager)

**Integration points:**
- Wire into `jcode login` flow — after OAuth completes, call `CredentialStore::create()`
- Wire into `jcode secrets` CLI — `jcode secrets set provider.<id>.api_key`

### Phase 2: Integration — OAuth Lifecycle (3-4 days)

**Goal:** Implement full OAuth attempt state machine + connection tracking.

**State machine (from opencode `integration.ts` `lines 128-150`):**
```
Attempt:
  status: pending | complete | failed | expired
  mode: auto | code
  time: { created, expires }  // 10-min TTL
  callback: Effect<Credential.Info>  (auto)  |  (code: String) → Effect<Credential.Info>  (code)

Flow:
  1. integration.oauth() → returns Attempt with URL + instructions
  2. Auto mode: open browser → callback server listens → on success → credential stored
  3. Code mode: return URL to user → user pastes code → credential stored
  4. Scrub timer: every 30s, expire stale attempts
```

**Connection detection (from opencode `integration.ts` `lines 344-369`):**
- OAuth method → check CredentialStore for saved credentials
- API Key method → user provides key → saves to CredentialStore
- Env method → scan process env for matching variable names
- `.connection.for(provider_id)`: returns `OAuth{label} | Key{id} | Env{name} | NotConfigured`

**Files to modify:**
- `crates/jcode-provider-service/src/integration.rs` — new implementation
- `crates/jcode-provider-service/src/attempt.rs` — OAuth state machine
- `crates/jcode-app-core/src/external_auth.rs` — refactor to use Integration trait
- `src/cli/login.rs` — refactor to delegate to Integration.oauth()

### Phase 3: Catalog — Dynamic Provider/Model Registry (3-4 days)

**Goal:** Replace hardcoded provider lists (`ProviderChoice` enum, `models.rs`) with a dynamic Catalog populated at boot.

**Provider registration at boot (from opencode `catalog.ts` `lines 135-200`):**
```
For each jcode-provider-* crate:
  1. At compile time: inventory::submit! with ProviderInfo metadata
  2. At boot: Catalog scans inventory, calls register_provider()
  3. For each model the provider supports: register_model()
  4. Integration scans registered providers → auto-detect env vars
```

**`available()` logic (from opencode `catalog.ts` `lines 96-101`):**
```
available(provider, integration, connected) =
  NOT provider.disabled
  AND (provider has inline API key
       OR connected  // via CredentialStore
       OR integration exists with env method that has the env var set)
```

**`default()` logic (from opencode `catalog.ts` `lines 262-280`):**
```
default_model =
  config.default_model              // user preference from config.toml
  ?? config.pinned_default          // from model_prefs.json
  ?? latest(available_models)       // sort by release date, pick newest
  ?? error("no available model")    // prompt user to connect a provider
```

**`small()` logic (from opencode `catalog.ts` `lines 282-327`):**
```
small_model = cheapest enabled model
  matching /nano|flash|lite|mini|haiku|small/
  filtered: cost > 0 AND age <= 18 months
  sorted: (cost/maxCost * 0.8 + age/maxAge * 0.2)
```

**Files to modify:**
- `crates/jcode-provider-service/src/catalog.rs` — Catalog trait implementation
- `crates/jcode-provider-service/src/registry.rs` — boot-time registration
- `crates/jcode-provider-core/src/models.rs` — refactor to delegate to Catalog
- `crates/jcode-provider-core/src/selection.rs` — refactor to use Catalog.available()

### Phase 4: CLI Rewrite — Provider/Model Commands (2-3 days)

**Goal:** Replace `--provider` flag + `ProviderChoice` enum with dynamic `jcode provider` + `jcode model` commands.

**New CLI behavior:**

| Command | Old behavior | New behavior |
|---------|-------------|-------------|
| `jcode provider list` | Hardcoded list from `ProviderChoice` enum | Catalog.available() with connection status + auth method hints |
| `jcode provider connect <id>` | Login flow per-provider | Integration.oauth() or prompt API key → saves to CredentialStore |
| `jcode provider disconnect <id>` | — | Removes credential from CredentialStore |
| `jcode model list` | Read from `models.rs` mapping | Catalog.models() with cost + capabilities + context window |
| `jcode model default <p> <m>` | — | Catalog.set_default() persisted to config |

**Flag changes:**
- `--provider` becomes **dynamic String** (not `ProviderChoice` enum). Resolves against Catalog.
- `--model` resolves against Catalog (not hardcoded `models.rs`)
- `--provider-profile` kept for OpenAI-compatible custom profiles

**Files to DELETE:**
- `src/cli/provider_init.rs` — `ProviderChoice` enum (replace with dynamic String ID)

**Files to MODIFY:**
- `src/cli/args.rs` — change `--provider` type from enum to `Option<String>`

### Phase 5: TUI Provider Manager (3-4 days)

**Goal:** Interactive provider/model management in TUI.

**New files:**

| File | Description | OpenCode Equivalent |
|------|-------------|-------------------|
| `crates/jcode-tui/src/tui/provider_picker.rs` | List providers, show connection status, connect/disconnect | dialog-connect-provider |
| `crates/jcode-tui/src/tui/model_picker.rs` | Enhanced model picker with cost + capabilities + search | dialog-select-model |

**`/provider connect` TUI flow:**
1. User types `/provider` or uses inline picker
2. Shows all providers grouped: connected | available | not configured
3. Select unconnected provider → shows auth methods
4. API Key: prompts for key → `Integration.connection.key()` → saves to CredentialStore
5. OAuth: opens browser → shows "waiting" spinner → on complete → CredentialStore saves → Catalog updates

**`/model` TUI flow (enhanced from current):**
1. User types `/model` or clicks model label in status bar
2. Section headers: Favorites > Recent > Connected Providers > All Providers
3. Each row: model name, cost per million tokens, context window, capability badges
4. `f` toggles favorite (persisted to `model_prefs.json`)
5. Enter selects model (and optionally sets default)
6. Connected providers' models shown first

### Phase 6: Session Runner Rewire (3-4 days)

**Goal:** Replace `Agent::new()` → `ActiveProvider` resolution chain with Catalog-based resolution.

**New resolution chain:**
```
Agent::new():
  1. Read config → --model | config.default_model | Catalog.default()
  2. Resolve model → Catalog.model.get(provider_id, model_id)
  3. Check credentials → Integration.connection_for(provider_id)
  4. Build 4-axis Route:
     Route {
       Protocol:   determine from provider (anthropic-messages / openai-chat / ...)
       Endpoint:   from provider.api.base_url + model.api.url
       Auth:       from connection info (Bearer token / API key header / sigv4 / ...)
       Framing:    HTTP SSE / WebSocket / AWS EventStream
     }
  5. Pass Route to Provider::new(Route)  // the OLD Provider trait wraps this
     OR: call protocol directly if fully migrated
```

**`/model` switch at runtime:**
```
  1. User picks new model → Catalog.model.get()
  2. Check if provider is still connected
  3. If not connected → show "please connect provider" dialog
  4. Build new Route
  5. Hot-swap provider in running session
```

**Provider failover (enhance):**
```
  On RateLimit/503 error:
  1. Classify error (rate-limit / quota / server-error / auth)
  2. If retryable → Catalog.provider.available().next()
  3. Build new Route for next provider
  4. Inject explanation into prompt: "Switched to ... because ..."
```

**Files to MODIFY:**
- `crates/jcode-app-core/src/agent.rs` — `Agent::new()` uses Catalog + Integration
- `crates/jcode-app-core/src/turn_execution.rs` — model switching uses Catalog
- `crates/jcode-agent-runtime/src/lib.rs` — session creation uses new resolution

## 6. Migration Risks

| Risk | Likelihood | Mitigation |
|------|-----------|------------|
| Old Provider trait consumers break | High | Keep old trait working until Phase 6. Phase 7 is the last step |
| All providers need updating | High | Phases 0-5 don't touch provider implementations. Only Phase 6 rewires them |
| Config backward compat | Medium | Config reader maps old `[provider]` block → new Catalog fields |
| User disruption | Medium | Keep `--provider` flag working throughout (dynamic resolution behind scenes) |
| OAuth callback server complexity | Medium | Start with API key flow (no callback needed). Add OAuth in Phase 2b |
